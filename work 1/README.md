# 计算机图像学作业

#### 介绍

![image](https://github.com/dogbear-china/Graphic_works/blob/main/work%201/%E6%BC%94%E7%A4%BA.gif)

本项目是计算机图像学课程作业，基于 Taichi 框架实现 GPU 并行加速的万有引力粒子群交互仿真，结合图形学实时渲染与物理引擎基础，实现粒子的引力运动、边界碰撞、鼠标交互等核心效果，完成图像学中并行计算与实时可视化的实践要求。

#### 软件架构

项目采用分层解耦的工程化架构设计，严格遵循 Python 模块规范，将配置层、GPU 计算层、视图渲染层分离，各模块职责单一、低耦合，符合计算机图像学中 “计算与渲染分离” 的设计思想，架构如下：

CG-Lab/

└── src/

    └── Work0/

        ├── \_\_init\_\_.py       # Python模块标识文件，声明包结构

        ├── config.py         # 配置层：集中管理物理/渲染所有参数，一键修改无需改动核心代码

        ├── physics.py        # GPU计算层：Taichi数据结构定义+核函数，仅负责物理数学计算（无渲染逻辑）

        └── main.py           # 视图渲染层：程序入口，负责Taichi初始化、鼠标交互、图形实时渲染

**各模块核心作用**

配置层（config.py）：抽离所有硬编码参数，包含粒子数量、引力强度、空气阻力、窗口分辨率、粒子颜色 / 半径等，实现参数热更新；

GPU 计算层（physics.py）：基于 Taichi 的@ti.kernel装饰器实现 GPU 并行计算，完成粒子初始化、引力计算、速度更新、边界碰撞检测等核心物理逻辑，仅关注数值计算，与渲染引擎解耦；

视图渲染层（main.py）：作为计算与渲染的桥梁，负责 Taichi 底层 GPU 接管、GUI 窗口创建、鼠标输入读取、显存数据读取与粒子图形渲染，调用计算层接口驱动物理更新。

#### 特技

1. 开发语言：Python 3.10+
2. 核心框架：Taichi 1.7.4（GPU 并行计算 + 实时图形渲染）
3. 环境管理：uv（轻量 Python 包 / 环境管理工具）
4. 图形渲染：Taichi 内置 GUI（ti.GUI）
5. 特技
6. 基于 Taichi 实现GPU 并行加速，将粒子物理计算任务分发至 GPU 核心，支持上万粒子的实时并行运算与渲染，体现计算机图像学的硬件加速思想；
7. 实现鼠标交互式引力源，鼠标位置作为动态引力中心，粒子随鼠标移动实时响应引力，结合空气阻力、边界反弹的物理效果，还原真实的粒子运动规律；
8. 严格的参数解耦与工程化设计，所有可调参数集中管理，无需修改 GPU 核函数与渲染代码，即可完成效果调优，便于后续功能扩展（如粒子颜色渐变、多引力源等）；
9. 遵循图形学计算与渲染分离原则，物理计算逻辑不依赖任何渲染接口，可无缝替换为其他渲染引擎（如 Pygame/Matplotlib），具备良好的扩展性；
10. 适配 Taichi 多架构运行，支持 GPU（CUDA/Metal/Vulkan）与 CPU 自动降级，兼容不同硬件环境，保证作业的可运行性。
